monitor.c
cvoid   *monitor_routine(void *arg);
    // Thread entry point (cast arg → t_sim *).
    // Polls every ~1 ms; for each coder checks:
    //   now_ms() - last_compile_start_ms > time_to_burnout
    // If burnout detected: log "X burned out", set stop_flag,
    //   pthread_cond_broadcast all dongle cvars to unblock waiters.

static int  all_done(const t_sim *sim);
    // Returns 1 if every coder->compile_count >= nb_compiles_required

simulation.c
cint     sim_init(t_sim *sim, const t_args *args);
    // Allocates coders[] and dongles[], inits mutexes,
    // seeds last_compile_start_ms = now_ms() for all coders.

int     sim_run(t_sim *sim);
    // pthread_create for monitor + all coder threads.
    // pthread_join in reverse order.
    // Returns 0 on clean stop, -1 on burnout.

void    sim_destroy(t_sim *sim);
    // Frees coders[], calls dongle_destroy on each dongle,
    // destroys log_mutex and state_mutex.

Key implementation notes
Dongle asymmetry — always acquire the lower-id dongle first to avoid deadlock
(the classic left-right fork ordering from dining philosophers).
The one-coder edge case is handled separately (only one dongle, no deadlock possible).
Burnout timer reset — last_compile_start_ms must be set to now_ms()
at the moment the coder starts compiling (after both dongles are held),
not when they request a dongle.
EDF tie-breaking — when two deadlines are equal,
break by coder_id (lower wins) to keep the policy fully deterministic.
Cooldown vs. scheduling — cooldown is checked inside
dongle_take before honoring a scheduling grant; a coder at the head of
the queue still waits out the cooldown with pthread_cond_timedwait.
